# Codex·Claude Code 스킬 사용 가이드 문서 설계

## 목적

ModelConvertor를 Codex 또는 Claude Code 스킬에서 호출할 때 사용자가 알아야 할 입력 방식, 누락 정보 처리, 출력 위치를 한 문서에서 설명한다. 사용자는 내부 JAR 명령이나 `--sql-file` 옵션을 직접 조합하지 않아도 된다는 점을 명확히 한다.

## 문서 구성

### 상세 가이드

새 문서 `docs/skill-usage-guide.md`를 스킬 사용법의 단일 원본으로 작성한다. 다음 내용을 포함한다.

- 사용자, Codex·Claude Code 스킬, ModelConvertor CLI의 역할 구분
- SQL 본문을 대화에 직접 입력하는 방법
- `.sql`, `.txt` 등 UTF-8 텍스트 파일 경로를 전달하는 방법
- 클래스명 또는 패키지명이 없을 때 스킬이 누락된 값만 되묻는 흐름
- 스킬이 값을 모두 확보한 뒤 JAR를 비대화형으로 실행한다는 설명
- 출력 경로를 생략했을 때의 기본 위치와 파일명
- `--stdout`, 사용자 지정 출력 경로, 기존 파일 덮어쓰기 동작
- 파일 미존재, 잘못된 SQL, 설정 또는 JDBC 드라이버 누락 시의 오류 처리 원칙
- 대표적인 사용자 요청 예시와 예상 대화

문서는 사용자가 입력할 자연어 요청을 중심으로 작성한다. 내부 명령은 동작을 설명하는 보조 예시로만 제공한다.

### README 안내

`README.md`에는 짧은 “Codex·Claude Code 스킬 사용” 섹션을 추가한다. 다음만 포함한다.

- 스킬을 통하면 SQL 본문 또는 텍스트 파일 경로를 자연어로 전달할 수 있다는 요약
- 클래스명과 패키지명이 없으면 스킬이 되묻는다는 설명
- 상세 가이드 링크

상세 동작과 예시는 README에 복제하지 않는다.

## 사용자 흐름

1. 사용자가 SQL 본문이나 파일 경로와 함께 DZ 모델 생성을 요청한다.
2. 스킬은 입력이 존재하는 파일 경로인지 SQL 본문인지 판단한다.
3. 클래스명과 패키지명 중 빠진 값이 있으면 사용자에게 해당 값만 질문한다.
4. 스킬은 완성된 옵션으로 ModelConvertor를 실행한다.
5. 출력 옵션이 없으면 현재 작업 디렉터리 아래 `src/main/java/{패키지 경로}/{클래스명}.java`에 생성한다.
6. 기존 파일이 있으면 임의로 덮어쓰지 않고 사용자에게 알린다.

## 정확성 기준

- 파일 확장자를 `.sql`로 제한한다고 설명하지 않는다. 현재 구현은 `--sql-file`로 받은 모든 UTF-8 텍스트 파일을 읽는다.
- 스킬과 사용자의 대화형 흐름과 JAR 프로세스의 비대화형 실행을 구분한다.
- 파이프 입력에서 클래스명과 패키지명이 없으면 JAR가 오류를 반환한다는 현재 동작을 명시한다.
- 종료 코드 `2`는 CLI 인자, 클래스명·패키지명, SQL 사전 검증, 중복 결과 라벨·Java 필드명 등 호출 및 입력 검증 실패에 사용한다고 명시한다.
- 종료 코드 `1`은 SQL 입력 I/O, 설정 로딩과 필수 설정 키 누락, Oracle 연결·JDBC 처리, Oracle 실행 중 SQL 실패, 출력 파일 충돌 등 처리 실패에 사용한다고 명시한다.
- SQL 오류는 애플리케이션의 사전 검증 실패면 종료 코드 `2`, Oracle/JDBC 처리 중 실패면 종료 코드 `1`로 구분한다.
- `--stdout` 사용 시 파일이 생성되지 않는다고 명시한다.
- 기본 출력 경로와 기존 파일 충돌 동작은 현재 구현 및 README와 일치시킨다.

## 범위 제외

- `.agents/skills` 또는 `.claude/skills` 아래 실제 스킬 생성

- 설치 자동화 및 Oracle JDBC 드라이버 배포

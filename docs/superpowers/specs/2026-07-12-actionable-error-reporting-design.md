# ModelConvertor 오류 메시지 개선 설계

- 작성일: 2026-07-12
- 대상: `modelconvertor`
- 상태: 승인됨

## 목적

설정 파일, SQL 파일, Oracle 연결 및 메타데이터 처리 실패 시 사용자가 실패 원인을 파악할 수 있는 메시지를 제공한다. 비밀번호는 원본 예외 메시지에 포함되어 있더라도 출력하지 않는다.

## 범위

### 포함

- 설정 로드 실패 시 설정 파일 경로와 원인 출력
- SQL 파일 읽기 실패 시 SQL 파일 경로와 원인 출력
- Oracle `SQLException` 발생 시 오류 코드와 안전하게 정리한 원인 출력
- Oracle 오류 메시지에서 설정된 비밀번호 제거
- 각 오류 흐름과 종료 코드를 검증하는 회귀 테스트
- 대화형 클래스명·패키지명 재입력을 추후 검토 문서에 기록

### 제외

- 대화형 재입력 구현
- 새로운 예외 클래스 계층
- 상세 로그 파일 또는 디버그 모드
- SQL 파서의 오류 위치 표시 개선

## 접근 방식

`ModelConvertorApplication.run`의 처리 경계에서 실패 단계별로 예외를 처리한다. 별도 도메인 예외를 추가하지 않고, 이미 알고 있는 입력 경로와 `SQLException` 정보를 이용해 메시지를 구성한다.

처리 순서는 다음과 같다.

1. SQL 입력 중 `IOException`이 발생하면 `SQL input failed: <경로>: <원인>`을 출력한다.
2. 설정 로드 중 `IOException`이 발생하면 `Oracle config failed: <경로>: <원인>`을 출력한다.
3. Oracle 연결·메타데이터 처리 중 `SQLException`이 발생하면 `Oracle processing failed (code <코드>): <원인>`을 출력한다.
4. 위 범주에 해당하지 않는 예외는 기존 `Processing failed` 메시지를 유지한다.

기본 설정 경로는 `OracleConfig.defaultPath()`로 계산한다. 표준 입력 실패처럼 SQL 파일 경로가 없는 경우에는 `standard input`을 대상으로 표시한다.

## 민감정보 보호

Oracle 오류 원문을 그대로 출력하지 않는다. 메시지에 현재 설정의 비밀번호와 동일한 문자열이 있으면 `***`로 교체한다. 비밀번호 자체, `oracle.password=<값>` 또는 `OracleConfig.toString()`의 원문을 출력하지 않는다.

빈 비밀번호는 필수 속성 검증에서 거부되므로 마스킹 함수는 빈 문자열을 교체 대상으로 사용하지 않는다.

## 종료 코드

| 실패 유형 | 종료 코드 |
|---|---|
| 설정 또는 SQL 파일 I/O 실패 | `1` |
| Oracle 연결·메타데이터 실패 | `1` |
| 잘못된 옵션·SQL·이름 | `2` |
| 예상하지 못한 처리 실패 | `1` |

기존 종료 코드 계약은 변경하지 않는다.

## 테스트

`ModelConvertorApplicationTest`에 다음 동작을 추가한다.

- 존재하지 않는 설정 파일의 절대 경로가 stderr에 포함된다.
- 존재하지 않는 SQL 파일의 절대 경로가 stderr에 포함된다.
- 주입한 `SQLException`의 오류 코드와 원인이 stderr에 포함된다.
- `SQLException` 메시지에 설정 비밀번호가 포함돼도 stderr에는 비밀번호가 없고 `***`가 표시된다.
- 기존 이름 검증, 출력 충돌, 성공 흐름 테스트가 계속 통과한다.

## 추후 검토

대화형 모드에서 클래스명 또는 패키지명이 비어 있거나 유효하지 않을 때 즉시 종료하지 않고 재입력받는 기능은 별도 개선 항목으로 기록한다. EOF 입력은 재시도하지 않고 종료하는 방향을 권장한다.
